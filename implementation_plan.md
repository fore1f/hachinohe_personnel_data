# Implementation Plan

## 概要

* 対象のプロジェクトパス: `c:\Users\tked1\py\hachinohe_personnel_data\`
* 使用する主要ライブラリ: `pandas`, `openpyxl`, `tkinter` (標準ライブラリ), `re`, `os`
* 実行環境: `uv` (Windows)

## スクリプトの構成 (`extract_personnel.py`)

1. **インポートとユーティリティ関数の定義**
    * `to_zenkaku`: 半角数字を全角数字へ変換する。
    * `format_retirement_date`: 「3月31日付け」などの退職日から「３１日」や「４月１日」のような文字列フォーマットを生成する処理。
    * `format_old_job`: 旧職（E列）のカッコを・に変換し、全体の先頭/末尾のみに「（）」を付与する。
    * `replace_brackets`: 全角半角のカッコ等、不要な記号を「・」に置換する。
    * `clean_str`: 列の値に対しNaNチェック、空白削除、「、」および「㈱」の除去や `replace_brackets` を一律適用する。
2. **メイン処理**
    * `tkinter.filedialog.askopenfilename()` によってWindowsのダイアログプロンプトを開き、処理対象のExcelファイル（`.xlsx` または `.xls`）を取得する。
    * `pandas.read_excel()` を用い、ヘッダ行なし (`header=None`) でExcelからDataFrame形式へ全データを読み込む。
3. **データ抽出ループ**
    * ヘッダ行の判定として、A列に「< 部長級 >」等の囲い記号があるかチェックする。級名と局名は都度更新し、局名が「市長事務部局」など前月から変更された場合は区切り行として出力する（同時に、以前のB列のキャッシュクリアも行う）。
    * A列（異動区分）が「退職」とそれ以外で分岐させる。
        * 「退職」の場合は、C列（新職の文字列）がある場合差し込み、独自フォーマットで整形する。
        * 「出向など（基本の異動区分以外）」の場合は、「[C列]・[A列]」となるようフォーマットする（ただし「降任（役職定年）」の場合は特例で「役職定年」と短縮する）。出向の場合はB列を結合しない。
        * 基本異動区分の場合は、直前に記憶した「B列（部名）」＋「C列（新職名）」のように出力する。ただし、級名が「部長級」「次長級」のいずれかであれば、事前に「B列」の変数（`display_bu`）を空にしておく。
    * 「再任用」や「採用」でE列の旧職が空文字となる場合は、E列情報として全角スペース（`　`）を差し込む。
4. **結果の出力処理**
    * リストに蓄積された処理結果を、UTF-8エンコーディングで `[元ファイル名]_抽出結果.txt` ファイルへ書き出す。
    * 書き込み完了後にコンソールへ「抽出が完了しました」と対象パスをプリントして終了する。
